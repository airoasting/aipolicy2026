# 인덱스 비교 격자 + 건수 지도 추가 — 설계 문서

- 날짜: 2026-06-07
- 대상: `index.html` (단일 파일 데이터 드리븐 사이트)
- 상태: 설계 승인 대기

## 1. 목적

현재 인덱스는 "읽는 카탈로그"다. 표지 → Executive Summary(산문) → 지역 스크롤 덱 순서로, 이 데이터셋의 가장 강한 사실인 **분포의 쏠림**(투자·인프라 65% vs 권익·분배 8%)을 산문 안에 묻어둔다. 독자가 16개 지역을 한눈에 비교하거나 "비어 있는 자리"를 시각적으로 확인할 수 없다.

이 작업은 **요약과 지역 덱 사이에 비교 도구 2개를 추가**해, 주장하던 쏠림·빈칸을 *보여준다*. 추가형이며 기존 화면은 손대지 않는다.

## 2. 범위

### 포함
- **① 비교 격자** (16지역 × 8카테고리 히트맵, 교차 점프)
- **② 건수 기반 간이 지도** (한국 SVG choropleth, 클릭 점프)

### 제외 (YAGNI / 후속 작업)
- C안(팩트체크 원장 정렬·필터 테이블) — 별도 작업
- 투자 *금액* 추출/정규화 — 지도는 **건수**만 사용
- 지역 스크롤의 '읽기 모드 강등' — 배치를 추가형으로 정해 스크롤은 그대로 유지
- `data.js` 수정 — 모든 수치는 `window.AI_DATA`에서 런타임 파생

## 3. 현행 구조 (확인된 사실)

- 데이터: `window.AI_DATA`(인라인 스크립트 내 `DATA`로 참조). 16개 지역, 각 `policies[]`.
- 지역 덱: 섹션 `id="r{i}"`, 정책 카드 `id="pol-{i}-{k}"` (i=지역 인덱스, k=지역 내 정책 인덱스, 데이터 순서).
- 카테고리 덱: 섹션 `id="cat{ci}"`, `data-cat` 속성 보유. `const CATS=['AI인프라','AI산업전환','AI신산업·R&D','행정혁신','인재·교육','시민일상·복지','안전·데이터권익','분배·공공성']`(index.html:894)에서 `catData`(ci↔CATS[ci]) 생성.
- 모드 토글: `mode` 변수 + `#modesw` 버튼(region/cat) + 좌측 `#regnav` 핀.
- 지도 자원: `kr_map.js` = `window.KR_MAP = {viewBox:"0 0 524 631", regions:[{id, key(한글 지역명), path(SVG d)}...]}`. **현재 index.html에 미연결.**
- 스크립트 로드: `<script src="data.js">`(769) → 인라인 앱 스크립트(770~). 덱은 `deck.innerHTML=...`(890), `catdeck.innerHTML=...`(900)에서 동기 렌더.
- 팔레트: `:root`의 `--paper #F1E6CB / --ink #3A2516 / --sub #7a6243` + 정당색(민주=`--blue #3F8BC4` 계열, 국민의힘=`--red #E5392A` 계열). `cssCls(party)`가 `ppp`/`min` 클래스 판정.

## 4. 컴포넌트 설계

### 4.1 비교 격자 (renderMatrix)

**역할**: 16×8 건수 행렬을 히트맵 `<table>`로 그리고, 칸/행/열 클릭으로 기존 덱의 정확한 위치로 점프한다.

**입력**: `DATA`(지역 순서 = 행, `#r{i}`와 일치), `CATS`(열 순서, 합계 내림차순이라 좌→우로 쏠림이 보임).

**파생값** (런타임 계산, 하드코딩 금지):
- `count(i, cat)` = `DATA[i].policies.filter(p=>p.category===cat).length`
- 행 합계 = `DATA[i].policies.length`, 열 합계 = Σ over i, 총계 = 106
- 캡션 비율: 좌 3열 합/총계, 우 2열 합/총계 → "왼쪽 3열 N% · 오른쪽 2열 M%" 자동 생성

**시각**:
- 밀도 4단계 warm scale: 0=종이(옅음, 비활성), 1=`#F6D9A8`, 2=`#EFA94A`, 3+=`#D2670C`. 기존 톤과 동일 계열.
- 행 라벨: 지역 약칭(기존 `SHORT` 맵 재사용) + 정당 dot(민주 파랑 / 국민의힘 빨강).
- 합계행/열은 `--ink` 배경으로 구분.
- 캡션에 "왼쪽 3열(투자) … / 오른쪽 2열(권익·분배) …" 명시.

**교차 점프 인터랙션**:
- **칸(건수≥1) 클릭** → region 모드 보장 → `k = DATA[i].policies.findIndex(p=>p.category===cat)` → `#pol-{i}-{k}`로 스크롤(기존 스크롤 헬퍼/`barH` 보정 재사용) → 카드에 `.flash` 클래스 1.2초 부여 후 제거.
- **칸(건수=0)**: `aria-disabled`, 클릭 비활성, 옅게.
- **행 라벨 클릭** → region 모드 → `#r{i}`.
- **열 헤더 클릭** → cat 모드 전환(기존 모드 토글 로직 재사용) → `ci = CATS.indexOf(cat)` → `#cat{ci}`. (인덱스 대신 `[data-cat="..."]` 쿼리로 점프해도 됨 — 더 견고)

**접근성**: `<table>` 시맨틱(thead/th scope). 클릭 칸은 `role="button"` + `tabindex="0"` + Enter/Space 핸들러 + `aria-label`("서울 · AI인프라 1건, 정책으로 이동").

**반응형**: ≤768px에서 가로 스크롤 컨테이너(`overflow-x:auto`) 또는 셀 폭 축소. 점(dot)·숫자 유지.

### 4.2 건수 기반 간이 지도 (renderCountMap)

**역할**: 한국 지도에서 지역별 공약 **건수**를 정당색 농도로 칠하고, 클릭 시 해당 지역으로 점프한다.

**입력**: `window.KR_MAP.regions`(path + 한글 key), `DATA`(key 매칭으로 건수·정당·당선자).

**매칭**: `KR_MAP.regions[].key` ↔ `DATA[].region` 정확 일치. 불일치 키는 콘솔 경고 후 회색 처리(데이터 드리븐 안전장치).

**시각**:
- 각 지역 path fill = 정당 base 색 + 건수에 따른 명도/채도(choropleth). 건수 많을수록 진하게.
- 라벨: 지역 약칭 + Σ건수(작은 폰트). 작은 면적(세종·광역시)은 라벨 생략 또는 외부 리더선 — 1차 구현은 생략 허용.
- 범례: 정당 2색 + 농도 = 건수 스케일.
- **편향 방어 주석**: "건수는 공약 *개수*이며 규모·실현 가능성이 아닙니다."

**인터랙션**: hover/focus → 툴팁(지역·당선자·정당·Σ건수). 클릭/Enter → region 모드 → `#r{i}` (i = `DATA.findIndex(d=>d.region===key)`).

**접근성**: 각 지역 `<path>` 또는 래퍼에 `role="button"` + `tabindex="0"` + `aria-label`.

## 5. 통합

1. `index.html:769` `<script src="data.js">` **다음 줄**에 `<script src="kr_map.js"></script>` 추가(인라인 앱 스크립트 이전). 주의: `kr_map.js`는 기존 `.gitignore`에서 "미사용"으로 제외돼 있었음 → 사용 자산이 되므로 해당 ignore 라인 제거하고 git 추적에 포함(본 설계 커밋에서 .gitignore 수정 완료).
2. `exsum` 섹션과 `drawerWrap` 사이에 컨테이너 2개 삽입:
   ```html
   <section id="matrix"><!-- renderMatrix가 채움 --></section>
   <section id="countmap"><!-- renderCountMap가 채움 --></section>
   ```
3. 인라인 앱 스크립트에 `renderMatrix()`, `renderCountMap()` 함수 추가. 덱 렌더(890/900) **이후**에 호출(점프 대상 `#pol-*`/`#r*`/`#cat*`가 DOM에 존재해야 함).
4. 모드 전환·스크롤 점프는 기존 헬퍼 재사용. 없으면 최소 래퍼(`goRegionAnchor(id)`, `goCatAnchor(id)`)만 추가하되 기존 `mode`/`#modesw` 상태와 동기화.
5. CSS는 기존 `:root` 변수만 사용. 새 색 토큰 도입 금지. `.flash` 하이라이트 키프레임 1개 추가 허용.

`data.js`, 표지(cover), 요약(exsum) 텍스트, 지역/카테고리 카드 마크업, 검색, 기존 내비.

## 7. 검증

- 로컬 서버(`node server.js`) → preview로 띄움.
- 격자: 임의 칸 클릭 → 정확한 `#pol-{i}-{k}`로 이동 + 플래시. 행 라벨 → 지역. 열 헤더 → 카테고리 모드+해당 섹션. 0건 칸 클릭 무반응.
- 지도: hover 툴팁, 클릭 → 지역 점프. 16개 지역 모두 색 채워짐(미매칭 키 0개).
- 반응형: 360px·768px·1200px에서 격자/지도 깨짐 없음.
- 콘솔 에러 0.

## 8. 리스크 / 결정

- **열 헤더→카테고리 점프**는 `CATS.indexOf` 인덱스보다 `[data-cat="..."]` 쿼리가 견고(카테고리 필터/검색으로 섹션 수가 달라질 가능성 대비).
- **작은 면적 지역 라벨**: 1차 생략 허용, 툴팁으로 보완. 욕심내지 않음(7점 처리 후 필요 시 9.5점).
- **지도 메시지 편향**(건수=규모 오해): 주석으로 방어. 그래도 약하면 후속에서 B를 금액형으로 승급하는 별도 작업.
- 격자/지도가 **데이터 드리븐**이라 정책·지역 수가 바뀌어도 자동 적응 → 재사용 자산.
